## 🔒 安全建议

### 短期 (1-2 周)

#### 1. 处理 protobuf 漏洞

**方案 A: 禁用受影响特性 (推荐)**

```toml
# 在默认特性中移除
[features]
default = ["basic-ai"]  # 移除依赖 rust-bert 和 llm-chain 的特性
```

**方案 B: 等待上游更新**

- 跟踪 rust-bert 和 llm-chain 的更新
- 这些库更新后及时升级

**方案 C: 使用 [patch] 强制升级**

```toml
[patch.crates-io]
protobuf = { version = ">=3.7.2" }
```

⚠️ 可能导致编译错误

#### 2. 更新 tantivy

- 等待 tantivy 0.26+ 发布，可能升级 lru 依赖
- 或使用 patch 强制 lru >=0.12.6

### 中期 (1 月)

1. **增加安全审计频率**
   - 每周运行 cargo audit
   - 订阅 RustSec 安全公告

2. **完善测试覆盖**
   - 为可选特性添加集成测试
   - 使用 `--all-features` 运行完整测试

3. **依赖精简**
   - 评估是否所有可选依赖都是必需的
   - 考虑移除长期未维护的依赖

### 长期 (3 月)
